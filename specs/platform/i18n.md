# Spec: Internacionalización (i18n)

**Estado:** DOCUMENTADO (ingeniería inversa del código)
**Ámbito:** `src/lib/i18n/*`
**Revisión:** 2026-06-08

---

## 1. Propósito

El sistema de i18n permite que la UI del escritorio (textos de chrome de
ventanas, etiquetas del CV, descripciones de apps, etc.) se muestre en tres
idiomas y se cambie en caliente desde un único conmutador de la barra de
tareas.

Toda la maquinaria vive en un solo archivo,
[`src/lib/i18n/config.ts`](../../src/lib/i18n/config.ts), y se apoya en un
**store Zustand propio** con persistencia en `localStorage`. **No** usa
`next-intl` (ver [§6](#6-next-intl-estado)).

Conviven dos mecanismos de traducción complementarios:

- **Strings de UI estáticas** → diccionarios en memoria (`UI`,
  `APP_DESCRIPTIONS`) indexados por `locale`.
- **Contenido localizado de datos** → objetos `{ es, en, hy }` embebidos en
  los datos (p. ej. `cv_data.json`), resueltos con `pickLocalized()` de
  [`src/lib/cv-data.ts`](../../src/lib/cv-data.ts).

## 2. Idiomas

El tipo `Locale` se define en
[`src/os/types.ts`](../../src/os/types.ts):

```ts
export type Locale = 'es' | 'en' | 'hy'
```

| Código | Idioma   | Bandera | Notas                          |
| ------ | -------- | ------- | ------------------------------ |
| `es`   | Español  | 🇪🇸      | Idioma por defecto y *fallback* |
| `en`   | Inglés   | 🇬🇧      |                                |
| `hy`   | Armenio  | 🇦🇲      | Alfabeto armenio nativo        |

`es` es el idioma inicial del store y el *fallback* universal en todas las
funciones de resolución.

> El mismo tipo `Locale` se redeclara de forma independiente en
> [`src/lib/ai.ts`](../../src/lib/ai.ts) (`'es' | 'en' | 'hy'`) para el
> backend de IA; no se importa desde `@os/types`, por lo que son definiciones
> paralelas que deben mantenerse sincronizadas a mano.

## 3. Store de locale (Zustand + persistencia)

Definido en
[`src/lib/i18n/config.ts`](../../src/lib/i18n/config.ts):

```ts
type I18nStore = {
  locale: Locale
  setLocale: (l: Locale) => void
}

export const useLocale = create<I18nStore>()(
  persist(
    (set) => ({
      locale: 'es',
      setLocale: (l) => set({ locale: l })
    }),
    { name: 'os:locale:v1' }
  )
)
```

- **Tecnología:** `zustand` + middleware `persist` (`zustand/middleware`).
- **Estado:** un único campo `locale` (inicial `'es'`).
- **Acción:** `setLocale(l)` reemplaza el locale activo.
- **Persistencia:** `localStorage`, clave **`os:locale:v1`**. El sufijo `v1`
  versiona el esquema de persistencia. Al recargar, el último idioma elegido
  se rehidrata automáticamente.
- El archivo lleva `'use client'`, así que el store es exclusivamente de
  cliente.

`zustand` está además en `optimizePackageImports` de
[`next.config.ts`](../../next.config.ts).

## 4. Estructura de strings

### 4.1 `UI` — etiquetas de interfaz

`UI: Record<Locale, Record<string, string>>`. Diccionario plano clave→texto,
una entrada por idioma. Cubre ~30 claves: controles de ventana (`open`,
`close`, `minimize`, `maximize`), estados (`loading`), formulario de contacto
(`send`, `name`, `email`, `message`, `messageSent`), flujo de quiz
(`quizStart`, `quizRetry`, `quizNext`, `quizPrevious`, `quizFinish`, `score`,
`correct`, `incorrect`, `unanswered`) y nombres de secciones/apps
(`experience`, `education`, `skills`, `projects`, `about`, `contact`, `quiz`,
`padel`).

Las tres ramas (`es`, `en`, `hy`) replican exactamente el mismo conjunto de
claves. `padel` es `'Padel Scout'` en los tres idiomas (no traducido a
propósito).

### 4.2 `APP_DESCRIPTIONS` — descripciones de apps

`Record<Locale, Record<string, string>>`. Indexado por **id de app** (`cv`,
`ai`, `projects`, `quiz`, `padel`, `contact`, `iaeks`, `linkedin`, `github`,
`about`). Texto largo descriptivo usado en los iconos del escritorio.

### 4.3 Función de traducción `useT()`

```ts
export function useT() {
  const locale = useLocale(s => s.locale)
  return (key) => UI[locale][key] ?? UI.es[key] ?? String(key)
}
```

Hook que devuelve un *traductor* `t(key)`. Cadena de *fallback*:
**idioma activo → español → la propia clave como string**. La firma tipa
`key` como `keyof (typeof UI)['es']`, de modo que solo se aceptan claves
existentes (autocompletado y chequeo en compilación).

### 4.4 Contenido de datos: `pickLocalized()`

Para strings que viven en los datos (no en `UI`), los valores se modelan como
objetos `{ es, en, hy }` y se resuelven con `pickLocalized(value, locale)` de
[`src/lib/cv-data.ts`](../../src/lib/cv-data.ts). Acepta también strings
planas (las devuelve tal cual) y aplica *fallback* `locale → es → en → primer
valor → ''`.

## 5. Consumo en UI

Consumidores reales del store/strings encontrados en el código:

| Archivo | Qué consume | Uso |
| ------- | ----------- | --- |
| [`src/os/Taskbar.tsx`](../../src/os/Taskbar.tsx) (`LocaleFlag`) | `useLocale` (`locale` + `setLocale`) | **Conmutador de idioma.** Botón de bandeja que cicla `['es','en','hy']` con `setLocale(order[(idx+1)%len])` y muestra la bandera `{ es:🇪🇸, en:🇬🇧, hy:🇦🇲 }`. |
| [`src/os/DesktopIcon.tsx`](../../src/os/DesktopIcon.tsx) | `APP_DESCRIPTIONS` + `useLocale` | `APP_DESCRIPTIONS[locale]?.[manifest.id] ?? manifest.description`. Descripción localizada del icono con *fallback* al manifest. |
| [`src/apps/cv/CvApp.tsx`](../../src/apps/cv/CvApp.tsx) | `useLocale` + `useT` | Etiquetas de pestañas/UI vía `t('loading')`, `t('experience')`, etc.; contenido del CV vía `pickLocalized(tr.x, locale)`. |
| [`src/apps/contact/ContactApp.tsx`](../../src/apps/contact/ContactApp.tsx) | `useT` | Etiquetas del formulario (`name`, `email`, `message`, `send`, `messageSent`). |
| [`src/apps/ai/AiApp.tsx`](../../src/apps/ai/AiApp.tsx) | `useLocale` | Selecciona sugerencias por idioma (`SUGGESTIONS_BY_LOCALE[locale]`) y **envía `locale` al backend** en el body de `POST /api/ai/chat` (`{ messages, locale }`) para que la IA responda en el idioma activo. |

Patrón de consumo: los componentes leen el slice con selector
(`useLocale(s => s.locale)`) o invocan `useT()`. Como Zustand re-renderiza a
los suscriptores al llamar `setLocale`, el cambio de idioma es **reactivo e
instantáneo** en toda la UI, sin recarga ni navegación.

> Los manifests de apps (`src/apps/*/manifest.ts`) también declaran texto, pero
> su localización efectiva en el escritorio se aplica vía `APP_DESCRIPTIONS`
> en `DesktopIcon`.

## 6. next-intl (estado)

**`next-intl` figura como dependencia (`^3.25.1`) en
[`package.json`](../../package.json) pero NO está cableado.** Evidencia
encontrada en el código:

- Cero imports de `next-intl` en `src/` o `app/`: no aparecen
  `useTranslations`, `NextIntlClientProvider`, `createTranslator`,
  `getRequestConfig`, `createMiddleware`, etc.
- [`next.config.ts`](../../next.config.ts) **no** envuelve la config con el
  plugin `createNextIntlPlugin()`.
- No existe `middleware.ts` (ni en raíz ni en `src/`), ni rutas con segmento
  `[locale]`, ni `i18n/request.ts`, ni catálogos `messages/*.json`.

**Conclusión:** la internacionalización funciona **íntegramente con el store
Zustand propio** + diccionarios en memoria. `next-intl` está instalado pero
inerte (dependencia muerta a efectos de runtime).

## 7. Notas / deuda técnica

- **`next-intl` sin uso.** Candidato a eliminarse del `package.json` o, si se
  pretende migrar, a cablearse de verdad. Hoy solo añade peso al árbol de
  dependencias.
- **Tipo `Locale` duplicado.** Definido en
  [`src/os/types.ts`](../../src/os/types.ts) y redeclarado en
  [`src/lib/ai.ts`](../../src/lib/ai.ts). Riesgo de divergencia si se añade un
  cuarto idioma; convendría una única fuente importada.
- **i18n solo en cliente.** El store lleva `'use client'` y persiste en
  `localStorage`; no hay resolución de idioma en servidor (sin SSR localizado,
  sin `lang` por ruta). El `<html lang>` no se sincroniza con el locale
  activo (no se halló código que lo actualice) → posible impacto SEO/a11y.
- **Cobertura parcial de claves.** `UI` cubre un set acotado de etiquetas;
  texto fuera de `UI`/`APP_DESCRIPTIONS`/datos `{es,en,hy}` puede quedar
  hardcodeado en un idioma (p. ej. `'Padel Scout'`, o el `title="Idioma: ..."`
  del conmutador en `Taskbar.tsx`).
- **Sin detección de idioma del navegador.** El arranque es siempre `es`
  salvo que exista valor persistido; no se lee `navigator.language` ni
  `Accept-Language`.
- **Versionado de persistencia manual.** La clave `os:locale:v1` no define
  `migrate`/`version` en el middleware `persist`; un cambio incompatible del
  esquema obligaría a bumpear el nombre a mano (`v2`).
